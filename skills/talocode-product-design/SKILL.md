# talocode-product-design

Design high-quality product interfaces without generic AI slop.

## Purpose

Help Talocode agents design polished, intentional product interfaces for:
- Tera learning UI
- Tradia trader dashboard
- ClientPad CRM
- WorkLane workspace
- Codra Canvas
- ClipLoop creator tools

## When to Use

Use this skill when:
- Building or redesigning product interfaces
- Creating landing pages or dashboards
- Designing UI components
- Improving visual hierarchy
- Reviewing interface quality

## Design Workflow

### 1. Define Product Mood

Before coding, establish:
- **Primary emotion**: Calm, energetic, focused, playful
- **Visual weight**: Light, medium, heavy
- **Color temperature**: Warm, cool, neutral
- **Spacing density**: Open, balanced, compact

### 2. Choose Visual Direction

Select a clear aesthetic:
- **Minimal**: Lots of white space, simple typography
- **Dense**: Information-rich, compact layout
- **Editorial**: Content-first, clean reading experience
- **Dashboard**: Data visualization, charts, metrics
- **Creative**: Bold colors, dynamic layouts

### 3. Define Layout Hierarchy

Structure content in clear levels:
- **Primary**: Main action or headline
- **Secondary**: Supporting information
- **Tertiary**: Metadata, labels, captions

### 4. Typography and Spacing

Choose consistent rules:
- Font size scale (14px base minimum for body)
- Line height (1.5-1.8 for readability)
- Spacing scale (4px, 8px, 16px, 24px, 32px)
- Max width for content (600-800px)

### 5. Interaction Quality

- Hover states for all interactive elements
- Focus states for keyboard navigation
- Transition durations (150-200ms)
- Click feedback
- Loading states

## Anti-Slop Rules

Avoid generic AI patterns:
- No random gradients
- No glowing borders
- No unnecessary animations
- No stock photos
- No placeholder text
- No feature lists without context
- No "modern" buzzwords without substance

## Design Review Checklist

Before shipping:
- [ ] Visual hierarchy is clear
- [ ] Typography is readable
- [ ] Spacing is consistent
- [ ] Colors have purpose
- [ ] Interactions feel intentional
- [ ] No visual clutter
- [ ] Mobile responsive
- [ ] Accessibility basics covered

## Implementation Checklist

Before coding:
- [ ] Mood defined
- [ ] Direction chosen
- [ ] Layout sketched
- [ ] Typography selected
- [ ] Colors defined
- [ ] Spacing scale set

## Notes

- Design is intentional, not accidental
- Every pixel should have a purpose
- Remove before adding
- Test on real devices
